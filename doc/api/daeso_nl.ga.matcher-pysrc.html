 54 -class Matcher(object): 

 55      """ 
 56      Graph matching for a pair of graphs 
 57       
 58      A Matcher provides a "match" method which reduces a weighed graph 
 59      alignment to a maximally-weighted graph matching by solving the linear 
 60      asignment problem. 
 61       
 62      The input is a predicted graph alignment. Formally, this alignment is a 
 63      weighted labeled bipartite graph, i.e.  
 64      - source and target nodes are in disjoint sets (the source and  
 65        target graphs)  
 66      - edges have a label (the alignment relation) 
 67      - edges have a weight (the classifiers' "confidence" associated with  
 68        the predicted relation) 
 69       
 70      The output is a graph matching. Formally, a matching is a bipartite graph 
 71      where     
 72      - source and target nodes are in disjoint sets, and 
 73      - the set of edges has no common nodes 
 74       
 75      In other words, every node has at most one edge. Notice that the matching 
 76      is not necessarily complete, that is, some nodes may remain unaligned. 
 77       
 78      In implementation terms, a graph alignment between a source graph with 
 79      source nodes s_1,...,s_N and a target graph with target nodes t_1, ..., 
 80      t_M is represented as a Numpy record array with N x M records. Each record 
 81      represents a possible node alignment, where  
 82      - the nodes are identfied by the "source_node_count" and  
 83        "target_node_count" fields, 
 84      - the predicted relation is present in the "pred_relation" field, and 
 85      - the associated weight is present in the "pred_weight" field.  
 86       
 87      The weights column is reshaped into a square matrix with dimensions equal 
 88      to the max(N,M). The weights are also converted to costs by subtracting 
 89      them from the maximum possible weight. This matrix is then input to a 
 90      solver which solves the linear assignment problem using the Hungarian 
 91      algorithm, linear programming, or any other algorithm. The solution (i.e. 
 92      a maximally weighted graph matching) is then inserted in the 
 93      "match_relation" field of the original record array, while alignments 
 94      below a certain threshold are ignored (i.e. not all nodes need to be 
 95      aligned) 
 96      """
