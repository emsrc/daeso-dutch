176          from the sentence:  
177          
178            "Verkiezingsposters van de Partij voor de Dieren in de provincie 
179            Zeeland worden systematisch vernield of gestolen ." 
180          
181          Empty nodes are omitted from the Dot graph and thus invisible to the 
182          annotator. Hence, when selecting the node "ppart" (id=26), the 
183          annotator expects to see the corresponding string "gestolen". 
184          However, according to the "begin" (=0) and "end" (=16) attributes, the 
185          corresponding string is the full sentence!  
186          
187          The reason is that "begin" and "end" attributes can only define a 
188          continous substring, and are thus unable to represent a gapped 
189          string. Futhermore, note that empty nodes can also occur in the 
190          middle of tree rather than at the start (as here) or end. Therefore, 
191          Alpino chooses to include the subtrings covered by empty nodes. 
192           
193          The solution is to ignore Alpino's "begin"/"end" attributes and to 
194          explicitly calculate the yield of each subtree. This means that the 
195          yield the above subtree becomes "gestolen", which is as intended. 
196          
197          So far so good. Now, consider the following example: 
198          
199            <node begin="12" cat="oti" end="18" id="22" rel="mod"> 
200              <node begin="12" end="13" id="23" pos="comp" rel="cmp" root="om" word="om" /> 
201              <node begin="13" cat="ti" end="18" id="24" rel="body"> 
202                <node begin="16" end="17" id="25" pos="comp" rel="cmp" root="te" word="te" /> 
203                <node begin="13" cat="inf" end="18" id="26" rel="body">

207                  </node>

211              </node>
